# 公司内部自助打印服务技术方案

## 1. 项目背景

公司现有一台闲置 Windows 笔记本和一台已接入办公网络的物理打印机。为了降低采购专用打印服务器的成本，并提升员工日常打印文件的便利性，计划将闲置笔记本改造为内部自助打印服务节点。

员工可通过浏览器或钉钉工作台访问打印页面，上传待打印文件，由服务端统一接收、排队、校验并调用本地 Windows 打印能力完成打印。

本方案定位为公司内部低成本打印服务，不替代专业商用打印管理系统。第一阶段优先保证功能稳定、安全可控和易于维护。

---

## 2. 建设目标

本项目目标如下：

1. 搭建一套公司内网可访问的自助打印服务。
2. 支持员工通过浏览器或钉钉工作台上传文件。
3. 服务端自动接收文件并加入打印队列。
4. 调用连接在 Windows 笔记本上的物理打印机完成打印。
5. 记录打印任务、打印状态、操作用户和错误日志。
6. 支持临时文件自动清理，避免敏感文件长期留存。
7. 支持后续扩展钉钉免登、打印审批、打印统计和多打印机管理。

---

## 3. 项目范围

### 3.1 第一阶段范围

第一阶段建议只支持 PDF 文件打印，原因如下：

1. PDF 格式稳定，打印版式可控。
2. Word、Excel 文件直接静默打印依赖 Office/WPS，容易出现打开失败、分页异常、字体缺失、打印区域错误等问题。
3. 统一使用 PDF 打印可以降低系统复杂度，提高成功率。
4. 后续可通过文件转换能力扩展 Word、Excel、图片等格式。

第一阶段功能范围：

| 功能              | 是否支持 |
| --------------- | ---- |
| PDF 文件上传        | 支持   |
| PDF 文件打印        | 支持   |
| 打印队列            | 支持   |
| 打印任务状态查询        | 支持   |
| 打印日志记录          | 支持   |
| 临时文件自动清理        | 支持   |
| 钉钉工作台入口         | 支持   |
| Word/Excel 直接打印 | 暂不支持 |
| 多打印机复杂策略        | 暂不支持 |
| 打印审批            | 暂不支持 |
| 打印计费            | 暂不支持 |

### 3.2 后续扩展范围

后续可根据实际使用情况扩展：

1. Word、Excel、图片自动转 PDF。
2. 钉钉免登，自动识别员工身份。
3. 部门权限控制。
4. 打印份数、单双面、黑白/彩色选择。
5. 打印统计报表。
6. 打印审批流程。
7. 多打印机调度。
8. 打印失败自动重试。
9. 管理后台。

## 4. 总体架构

系统采用轻量级内网部署架构：

```text
员工电脑 / 手机 / 钉钉
        ↓
浏览器访问打印页面
        ↓
Python Web 服务
        ↓
任务队列与任务数据库
        ↓
PDF 打印引擎
        ↓
Windows 打印服务
        ↓
物理打印机
```

推荐架构如下：

```text
+-------------------+
| 员工设备           |
| 浏览器 / 钉钉       |
+---------+---------+
          |
          | HTTP / HTTPS
          |
+---------v---------+
| Windows 笔记本     |
| Web 服务           |
| 打印任务队列        |
| SQLite 数据库       |
| 文件存储目录        |
+---------+---------+
          |
          | SumatraPDF / Windows Print Spooler
          |
+---------v---------+
| 物理打印机          |
| USB / 网络连接      |
+-------------------+
```

## 5. 技术选型

### 5.1 后端服务

推荐使用：

* Python 3.10+
* Flask 或 FastAPI
* Waitress 作为 Windows 下的生产级 WSGI 服务
* SQLite 作为第一阶段任务数据库
* pywin32 获取打印机信息
* SumatraPDF 作为 PDF 静默打印工具

第一阶段建议使用 Flask，原因是开发简单、部署成本低，适合内部小型工具。

### 5.2 打印引擎

不建议直接依赖 Windows 文件默认打开程序进行打印。

推荐使用 SumatraPDF 命令行打印 PDF：

```bash
SumatraPDF.exe -print-to "打印机名称" -silent "文件路径.pdf"
```

优势：

1. 对 PDF 打印支持稳定。
2. 支持命令行静默打印。
3. 不需要打开可见窗口。
4. 比系统默认关联程序更可控。
5. 部署简单。

### 5.3 数据库

第一阶段使用 SQLite 即可。

需要记录：

1. 打印任务 ID。
2. 上传用户。
3. 原始文件名。
4. 服务端保存文件名。
5. 打印机名称。
6. 任务状态。
7. 错误信息。
8. 创建时间。
9. 开始打印时间。
10. 完成时间。

后续如果并发量增加，可迁移到 PostgreSQL 或 MySQL。

## 6. 核心业务流程

### 6.1 打印流程

```text
员工打开打印页面
        ↓
选择 PDF 文件
        ↓
选择打印机
        ↓
上传文件
        ↓
服务端校验文件类型和大小
        ↓
生成唯一任务 ID
        ↓
保存文件到临时目录
        ↓
写入任务数据库
        ↓
加入打印队列
        ↓
后台打印线程消费任务
        ↓
调用 SumatraPDF 提交打印
        ↓
更新任务状态
        ↓
记录日志
        ↓
定时清理临时文件
```

### 6.2 任务状态设计

打印任务建议包含以下状态：

| 状态        | 说明                |
| --------- | ----------------- |
| pending   | 已上传，等待打印          |
| printing  | 正在提交打印            |
| submitted | 已提交到 Windows 打印队列 |
| success   | 打印命令执行成功          |
| failed    | 打印失败              |
| deleted   | 临时文件已清理           |

说明：

`success` 不一定代表纸张已经完全打印完成，只代表服务端已经成功把打印任务提交给 Windows 打印系统。若要精确获取打印机真实完成状态，需要进一步对接 Windows Print Spooler 队列。

## 7. 文件处理方案

### 7.1 文件类型限制

第一阶段只允许上传 PDF 文件：

```text
允许格式：.pdf
不允许格式：.doc、.docx、.xls、.xlsx、.jpg、.png
```

后续如需支持 Office 文件，应先将文件转换为 PDF，再走统一打印流程。

### 7.2 文件大小限制

建议限制单个文件大小：

```text
默认最大：50MB
可配置范围：10MB - 100MB
```

避免大文件导致服务阻塞、磁盘占满或打印机长时间卡死。

### 7.3 文件命名规则

禁止直接使用用户上传的原始文件名作为服务端文件名。

推荐格式：

```text
uploads/
  2026-06-16/
    job_20260616_153012_a8f3c9.pdf
```

数据库中保留原始文件名，服务端存储使用唯一文件名。

### 7.4 临时文件清理

推荐清理策略：

| 文件状态        | 清理策略          |
| ----------- | ------------- |
| 打印成功        | 10 分钟后删除      |
| 打印失败        | 保留 24 小时，便于排查 |
| 超过 24 小时文件  | 自动删除          |
| 无数据库记录的孤立文件 | 每日清理          |

## 8. 钉钉集成方案

### 8.1 第一阶段集成方式

第一阶段采用钉钉工作台 H5 应用方式集成。

管理员在钉钉后台创建企业内部应用，将应用首页地址配置为打印服务地址。

示例：

```text
http://print.company.local
```

或在没有内网域名的情况下使用固定 IP：

```text
http://192.168.1.100:5000
```

更推荐使用内网域名，避免笔记本 IP 变化导致服务不可访问。

### 8.2 网络要求

员工设备必须能够访问打印服务所在笔记本。

适用场景：

1. 员工电脑与打印服务器在同一公司局域网。
2. 员工手机连接公司 Wi-Fi。
3. 公司网络允许访问打印服务器端口。

如果员工使用手机流量或外部网络，则无法直接访问内网 IP。

### 8.3 第二阶段钉钉免登

如果需要识别员工身份，应接入钉钉免登能力。

免登后可实现：

1. 自动识别当前登录员工。
2. 记录打印人。
3. 按部门控制打印权限。
4. 统计个人或部门打印量。
5. 支持审批流程。

第一阶段如果不接免登，至少应设置一个简单的访问密码或内网访问白名单，避免所有同网段用户都可以随意打印。

## 9. 安全设计

### 9.1 访问控制

第一阶段至少需要实现以下安全措施：

1. 仅允许公司内网访问。
2. 固定服务端口。
3. Windows 防火墙只开放必要端口。
4. 可选增加简单登录密码。
5. 禁止匿名公网访问。

第二阶段建议接入钉钉身份认证。

### 9.2 文件安全

需要控制上传文件风险：

1. 只允许 PDF。
2. 限制文件大小。
3. 校验文件扩展名和 MIME 类型。
4. 文件名重命名，避免路径穿越。
5. 上传目录禁止执行脚本。
6. 打印后自动删除文件。
7. 记录上传人和打印时间。

### 9.3 日志审计

系统需要记录：

1. 谁上传了文件。
2. 上传了什么文件。
3. 使用哪台打印机。
4. 打印时间。
5. 打印状态。
6. 打印失败原因。
7. 文件清理时间。

日志可保留 30 天到 90 天，具体根据公司合规要求决定。

## 10. 部署方案

### 10.1 Windows 笔记本准备

硬件要求：

1. Windows 10 或 Windows 11。
2. 稳定电源，建议长期接电。
3. 关闭自动睡眠。
4. 打印机驱动已正确安装。
5. 能正常通过系统打印测试页。
6. 设置固定 IP 或 DHCP 地址保留。

### 10.2 软件安装

需要安装：

1. Python 3.10+
2. SumatraPDF
3. 项目依赖包
4. NSSM 或 WinSW，用于注册 Windows 服务

Python 依赖示例：

```bash
pip install flask waitress pywin32
```

### 10.3 服务运行方式

开发测试阶段可以使用：

```bash
python app.py
```

正式运行不建议直接使用该方式。

正式部署建议：

```text
Waitress + NSSM + Windows 开机自启
```

部署方式：

1. 使用 Waitress 启动 Web 服务。
2. 使用 NSSM 注册为 Windows 服务。
3. 设置服务开机自启。
4. 输出日志到指定目录。
5. 配置 Windows 防火墙入站规则。
6. 设置笔记本不休眠。

### 10.4 推荐目录结构

```text
print-service/
├── app.py
├── config.py
├── requirements.txt
├── data/
│   └── print_service.db
├── uploads/
│   └── 2026-06-16/
├── logs/
│   └── app.log
├── static/
├── templates/
└── tools/
    └── SumatraPDF.exe
```

## 11. 运维监控

### 11.1 日常检查项

管理员需要定期检查：

1. 服务是否正常运行。
2. 打印机是否在线。
3. 打印机是否缺纸。
4. 打印机是否卡纸。
5. 磁盘空间是否充足。
6. 上传目录是否正常清理。
7. 错误日志是否异常增多。

### 11.2 健康检查接口

建议提供健康检查接口：

```text
GET /health
```

返回内容：

```json
{
  "status": "ok",
  "printer_count": 1,
  "queue_size": 0,
  "disk_free": "120GB"
}
```

### 11.3 管理后台

后续可以增加简单管理后台，支持：

1. 查看任务列表。
2. 查看失败任务。
3. 重新打印。
4. 删除任务。
5. 查看打印机状态。
6. 导出打印记录。

## 12. 风险与应对

| 风险            | 影响       | 应对方案            |
| ------------- | -------- | --------------- |
| 笔记本休眠         | 服务不可用    | 关闭睡眠，长期接电       |
| IP 地址变化       | 钉钉入口无法访问 | 固定 IP 或配置内网域名   |
| 打印机离线         | 打印失败     | 增加状态检测和错误提示     |
| 文件过大          | 服务卡顿     | 限制上传大小          |
| Office 文件打印异常 | 版式错乱或失败  | 第一阶段只支持 PDF     |
| 员工误传敏感文件      | 数据泄露风险   | 打印后自动清理         |
| 无权限控制         | 被滥用      | 接入钉钉免登或访问密码     |
| 服务异常退出        | 无法打印     | 使用 Windows 服务守护 |
| 磁盘占满          | 服务不可用    | 定时清理和磁盘监控       |

目标：跑通 PDF 自助打印闭环。

工作内容：

2. 实现 PDF 上传。
3. 实现打印机列表读取。
4. 实现打印任务队列。
5. 使用 SumatraPDF 静默打印。

7. 增加任务状态查询。
8. 增加文件自动清理。
9. 配置 Windows 防火墙。
10. 在内网浏览器中完成测试。

验收标准：

1. 员工可打开打印页面。
2. 可上传 PDF。
3. 可选择打印机。

6. 打印失败可记录错误。

### 第二阶段：钉钉入口集成

工作内容：

2. 配置应用首页地址。

1. 员工可从钉钉工作台打开打印页面。
2. 手机连接公司 Wi-Fi 后可访问服务。
3. 非授权用户无法随意访问。
4. 打印任务可正常提交。

### 第三阶段：身份与审计

目标：具备基础企业级管理能力。

工作内容：

4. 增加部门权限控制。

工作内容：

2. Excel 转 PDF。

4. 增加转换失败提示。
5. 增加打印预览。

## 14. 验收标准

第一阶段上线前必须满足以下条件：

1. 服务能连续运行不少于 3 天。
2. PDF 打印成功率达到 95% 以上。
3. 打印失败时能记录错误原因。
4. 上传文件不会出现同名覆盖。
5. 上传目录可以自动清理。
6. 单文件大小限制生效。
7. 非 PDF 文件无法上传。
8. 打印任务有状态记录。
9. 笔记本重启后服务能自动启动。
10. 员工可在公司内网稳定访问。

## 15. 最终建议

本项目技术上可行，适合以低成本方式解决公司内部轻量级自助打印需求。

但需要注意，打印服务涉及文件上传、员工身份、敏感文档和硬件稳定性，不能仅以简单 Flask 上传页面直接上线。建议采用分阶段实施策略：

第一阶段先实现 PDF 自助打印 MVP，保证打印链路稳定。

第二阶段接入钉钉工作台，解决员工访问入口问题。

第三阶段接入钉钉免登和审计日志，满足内部管理要求。

第四阶段再扩展 Word、Excel、图片等格式。

整体推荐路线：

```text
PDF 打印 MVP
    ↓
钉钉工作台入口
    ↓
钉钉免登与审计
    ↓
Office 文件转换
    ↓
打印审批与统计
```

该路线可以降低初期开发风险，避免一开始就处理复杂的 Office 打印兼容问题，同时保证系统具备后续扩展能力。
